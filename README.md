# Custom Dashboard

A robust, reactive, and responsive Flutter administration dashboard. It supports managing dynamic sections of a client-facing application, including **Articles**, **Services**, **Categories**, **About Us Info**, and **Media Files**.

The project features a decoupled repository pattern allowing seamless switching between **Fake/Local Mock data** and **Real Firebase & Supabase services**.

---

## 🚀 Key Features

*   **Authentication**: Firebase-backed user authentication flow with robust route guards preventing unauthorized access.
*   **Articles Management**: Full CRUD operations for article entities, including automatic categorization, and paginated dynamic scroll retrieval.
*   **Services Management**: Management of system services with robust form validation using `Reactive Forms`.
*   **Categories Management**: Direct category management dialogs mapping to parent entity configurations.
*   **About Us Page**: Dynamic visual information editor featuring `FormArray`-based item lists, vision/mission configuration, and image replacement.
*   **Unified Media & File Library**: Fully integrated file upload system utilizing **Supabase Storage** for hosting and **Cloud Firestore** for media metadata tracking. Supports drag-and-drop file imports on supported platforms.
*   **Responsive layout**: Multi-platform support (Web, Desktop, and Mobile resolutions) using `ResponsiveBuilder` and fluid adaptive shell wrappers.

---

## 🛠️ Architecture & Tech Stack

The application conforms to standard, highly scalable architectural patterns:

*   **State Management**: [Riverpod](https://pub.dev/packages/flutter_riverpod) is used for reactive state management. Dynamic dashboard forms utilize Riverpod's `FamilyNotifier` to prevent asynchronous initialization race conditions during form binding.
*   **Form Validation**: [Reactive Forms](https://pub.dev/packages/reactive_forms) for structured, reactive state binding and strict validation control.
*   **Routing**: [GoRouter](https://pub.dev/packages/go_router) with redirection rules handling auth states and initial deep links.
*   **Internationalization**: Fully localized UI elements utilizing the `flutter_intl` code generator.
*   **Themes & Styling**: Built with Material Design 3. Custom themes (Light/Dark support) are centralized inside `ColorManager` and `AppTheme`.

## ⚙️ Environment Configuration

The application implements a dynamic Dependency Injection layer. By default, it operates in a **mock local dev mode** (`dev`) so that it can be run without setting up Firebase or Supabase configurations locally.

To toggle between development mock files and production Firebase & Supabase services, compile using the `env` dart define argument:

*   **Local Development (Fakes)**: Default setting.
    ```bash
    flutter run
    ```
*   **Real Backend (Firebase & Supabase)**:
    ```bash
    flutter run --dart-define=env=real
    ```

## 🏃 Getting Started

### 📋 Prerequisites

*   [Flutter SDK](https://docs.flutter.dev/get-started/install) (Ensure `flutter doctor` passes successfully)
*   For running the `real` environment:
    *   Firebase project configured with Firestore, Auth, and Firebase configurations updated in `lib/firebase_options.dart`.
    *   Supabase project with storage buckets active for media uploads (endpoint configure in `lib/main.dart`).

### 🔧 Setup Steps

1.  **Clone the Repository**
    ```bash
    git clone https://github.com/SaifGadalla/custom_dashboard.git
    cd custom_dashboard
    ```

2.  **Fetch Dependencies**
    ```bash
    flutter pub get
    ```

3.  **Run Code Generators** (if modifying localized strings or generating new annotations)
    ```bash
    dart run build_runner build --delete-conflicting-outputs
    ```

4.  **Run the Project**
    ```bash
    # Run in mock dev mode
    flutter run
    ```
